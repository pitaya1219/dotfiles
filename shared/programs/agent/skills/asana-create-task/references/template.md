# タスクテンプレート（Elec Hub）

Asana タスク説明欄の標準構成。各セクションはこの順序で `html_notes` に出力する。
中身は対話で埋める。空のセクションは残さず、質問して埋めるか、明示的に「なし」と記載する。
見出しは `<h2>背景</h2>` のように出す（`html-rules.md` 参照）。

## 背景
なぜこのタスクが必要か。元タスク・依頼元・経緯。関連 Asana タスクは `<a data-asana-gid="GID"/>` でリンクする。

## Ready条件
着手可能になるための前提条件（このタスクを始めてよい状態）。

## 受入条件
完了と判断するための条件。チェック可能な箇条書きで書く。

## 対象のリポジトリ
変更対象のリポジトリ。略称指定 → フル URL に解決する（SKILL.md Step 4）。

## 実働テスト・結果
実働での確認観点。少なくとも以下を明示する:
- 出力結果の確認（期待する出力・挙動）
- PM への連携（要否・タイミング）
- テーブル変更の有無（DB スキーマ変更があるか）

## 相談先の担当者
不明点が出た時の相談先。名前 / ロール。

## 資料・関連するSlackスレッド
参照資料 URL、Slack / Teams スレッド。外部 URL は `<a href="URL">表示テキスト</a>`（`&`→`&amp;`）。
