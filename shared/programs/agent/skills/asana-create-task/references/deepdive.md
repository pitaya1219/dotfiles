# Deepdive — 種別別の追加質問（トリガー節の実体。使い方は `SKILL.md` Step 5、追加手順は `../MAINTAINING.md`）

## セキュリティグループ / 公開設定の是正
- **トリガー**: セキュリティグループ, SecurityGroup, SG, 是正, 全ポート, 全IP, 0.0.0.0/0, インバウンド公開
- **追加質問**:
  1. 対象リソースの一覧（リソース名 / SG ID / 現状のルール / あるべきルール）を整理したい。列挙してもらえるか？
  2. 例外的に公開を維持すべきポート / IP はあるか？（あれば理由も）
  3. 変更による既存疎通への影響範囲（依存サービス）は？
- **補強先**: 「受入条件」「実働テスト・結果」
- **注意**: リソース一覧は表にしたくなるが、`html_notes` にテーブルタグは無い。`<ul><li>` で「リソース: 現状 → あるべき」の形に落とす（`html-rules.md` 参照）
