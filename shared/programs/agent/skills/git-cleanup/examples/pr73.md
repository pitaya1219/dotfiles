# Example: PR #73 Commit Cleanup

This is the actual commit cleanup we performed on VisionPipe PR #73.

## Original State

6 commits in `feat/queue-level-parallelism`:

```
69361e5 fix: Display all processing items in queue UI
d503e8d feat: Add dynamic memory-based concurrency control and settings UI
55a2f8d fix: Start multiple jobs in single startProcessing call
9925400 fix: Address critical bugs in queue-level parallelism
740d22b test: Fix ImageQueueManagerTest to avoid mocking issues
54b3209 feat: Implement queue-level parallelism for concurrent image processing
```

**Problem**:
- Too many small fix commits
- parallel.log accidentally committed
- Hard to review the logical progression

## Goal

Reorganize into 3 logical commits:
1. Core implementation + all bug fixes
2. Memory management feature
3. UI improvement

## Execution Using the Skill

### Method 1: Using Helper Script (Easiest)

```bash
cd VisionPipe

# Step 1: Analyze
git log --oneline main..HEAD

# Step 2: Create backup (script does this automatically)
./scripts/git-cleanup-commits.sh

# In interactive mode:
# Select option 1: Squash all commits into one
# Then manually split as needed
```

### Method 2: Manual Commands (What we actually did)

```bash
# 1. Create backup
git branch backup/feat-queue-level-parallelism

# 2. Squash commits 1-4 into one
GIT_SEQUENCE_EDITOR="sed -i '2,4s/^pick/fixup/'" git rebase -i main

# Result: 3 commits (54b3209+fixes, d503e8d, 69361e5)

# 3. Split commit 2 to separate UI changes
git reset --soft HEAD~1
git reset HEAD app/src/main/java/com/pitaya/visionpipe/queue/QueueViewActivity.kt

# 4. Commit memory management only
git commit -m "feat: Add dynamic memory-based concurrency control"

# 5. Commit UI changes
git add app/src/main/java/com/pitaya/visionpipe/queue/QueueViewActivity.kt
git commit -m "fix: Display all processing items in queue UI"

# 6. Verify parallel.log is not included
git log --name-status main..HEAD | grep parallel.log
# (should return nothing)

# 7. Update commit 1 message
GIT_SEQUENCE_EDITOR="sed -i '1s/^pick/reword/'" git rebase -i main
# (add detailed description)

# 8. Final verification
git log --stat main..HEAD
git diff origin/feat/queue-level-parallelism HEAD --stat

# 9. Push
git push --force-with-lease origin feat/queue-level-parallelism
```

## Lessons Learned

1. **Always backup first** - We created `backup/feat-queue-level-parallelism`
2. **Verify at each step** - Used `git log --oneline` frequently
3. **Check for unwanted files** - Found and removed parallel.log
4. **Use --force-with-lease** - Safer than plain --force
5. **Document well** - Detailed commit messages help reviewers
